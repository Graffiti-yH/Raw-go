# 📸 RAW 相机照片一键智能整理工具 (最终修复版)

这是一个专门为摄影师、数字资产管理者开发的自动化相机 RAW 格式文件整理脚本。它能够智能读取照片的 Exif 拍摄元数据，彻底重构杂乱的图片存储目录，自动将其规整为标准的 **“日期文件夹 + 规范化命名”** 格式。

特别解决了多卡、多批次无序导出照片时，**重复运行脚本导致重名覆盖或二次打乱** 的行业痛点。

---

## ✨ 核心功能亮点

1. **智能增量识别（防重复处理）：** 自动通过正则算法匹配照片所在的目录格式和文件名。如果照片已经处于正确的日期文件夹且命名规范（如 `20260523_0001.arw`），脚本将自动跳过，绝不进行二次移动。
2. **尼康物理保护解锁：** 自动查找并解除部分尼康相机（.nef）因机内锁定（如保护照片不被误删）而被系统添加的“硬锁定标记”（uchg 与只读权限），确保文件可以被安全地移动与改名。
3. **全自动化扁平化提取：** 深度递归扫描子文件夹，将所有未整理的杂乱照片统一提取到根目录进行集中处理，同时在处理完毕后自动清理空文件夹，不留垃圾目录。
4. **高精度 Exif 时间轴排序：** 依赖专业的 `exiftool` 工具，精准提取照片真实的拍摄时间（DateTimeOriginal / CreateDate），以此为依据对每个日期文件夹内的照片按拍摄先后顺序从 `0001` 开始重命名。
5. **文件防覆盖机制：** 移动与重命名时伴随极其严格的重名检测。若遇到相同时间或相同原始文件名的照片，脚本会自动加后缀（如 `_1`, `_2`），确保摄影师珍贵的原生 RAW 资产 100% 安全。

## 📋 技术依赖与支持

### 1. 操作系统环境
* **最佳支持：** macOS（因涉及 `chflags nouchg` 指令，脚本在 Mac 环境下对尼康锁定的解除最完美）。
* **基本支持：** Linux / Windows (Git Bash)。需要在标准类 Unix 或模拟类 Unix 终端下运行。

### 2. 核心底层依赖
脚本高度依赖名为 **`exiftool`** 的开源图像元数据读写工具。如果系统未安装此工具，脚本将无法获取拍摄时间，从而导致整理失败。
* **macOS 安装：** `brew install exiftool`
* **Ubuntu / Debian 安装：** `sudo apt-get install libimage-exiftool-perl`
* **Windows 安装：** 需前往官网下载 `exiftool.exe` 并将其路径配置到系统环境变量中。

### 3. 支持的 RAW 文件格式
脚本默认识别并处理以下主流相机厂商的 RAW 格式：
* 佳能：`.cr3`, `.cr2`
* 尼康：`.nef`
* 索尼：`.arw`
* 大疆/通用：`.dng`
* 富士/徕卡等：`.raw`, `.orf`, `.rw2`

## 🛠️ 脚本运行五大核心步骤

当你在照片根目录下启动该脚本时，它会严格按照以下逻辑和顺序进行流水线式处理：
1. **🔓 1/5 解除尼康锁定文件：** 清除文件可能存在的系统不可修改旗标并赋予写权限，防止后续移动文件时报错。
2. **📂 2/5 扁平化目录：** 将嵌套在深层子文件夹中的所有未整理照片全部提取到当前根目录下，同时通过正则屏蔽，绝对不会影响已经属于 `YYYY-MM-DD` 格式文件夹内的任何文件。
3. **🧹 3/5 清理备份文件：** 自动清理以前由于运行其它照片修改软件可能遗留的 `*_original` 临时或垃圾备份文件，释放磁盘空间。
4. **🔄 4/5 整理 RAW 文件：** * **按日期归类：** 读取照片内部拍摄日期，自动建立形如 `2026-05-23` 的文件夹，并将照片移入对应的日期目录中。
   * **严格时序重命名：** 深入到每个日期文件夹中，将内部所有照片按精确到秒的拍摄时间先后排序，统一重命名为：`YYYYMMDD_序号.后缀`（如：`20260523_0001.arw`）。
5. **📦 5/5 移动非 RAW 文件：** 将根目录下所有不属于摄影 RAW 格式的文件（如相机附带的日志、不小心混入的文档等）全部迁移到统一自动创建的 `/其他` 文件夹中，保持核心工作流的绝对纯净。
